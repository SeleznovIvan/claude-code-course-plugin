## v1.1.0 ‚Äî Teaching Modes & Student Feedback

<Aside type="tip">Latest release</Aside>

### Teaching Modes

3 selectable learning styles ‚Äî choose when you start the course:

| Mode | Philosophy | Best For |
|------|-----------|----------|
| **Sensei** | "I show you the path. You walk it yourself." | Experienced devs who want muscle memory |
| **Coach** | "I'll guide you, but I'm here if you need a hand." | Most learners (default) |
| **Copilot** | "Let's build this together." | Beginners or time-constrained |

Mode affects how the instructor handles the ACTION phase, hints, and "stuck" situations. Verification is identical across all modes.

### Security: permissions.deny

Replaced `.claudeignore` (which never existed in Claude Code) with the official `permissions.deny` mechanism in `.claude/settings.json`:

```json
{
  "permissions": {
    "deny": [
      "Read(./.env)",
      "Read(./.env.*)",
      "Read(./secrets/**)"
    ]
  }
}
```

### Hooks Accuracy

- Fixed matcher format to **string regex** (`"matcher": "Write|Edit"`) ‚Äî verified against [official Anthropic repo](https://github.com/anthropics/claude-code/blob/main/plugins/plugin-dev/skills/hook-development/SKILL.md)
- Fixed HTTP hook timeout default (600s ‚Üí 30s)
- Added reference to official `hook-development` skill in plugin-dev plugin
- Prompt-based hooks now recommended over command hooks
- Added 4 missing events: StopFailure, PostCompact, Elicitation, ElicitationResult

### Student Feedback Fixes

- "Never skip failed tasks" rule ‚Äî Claude won't mark steps complete if they didn't work
- Commands restart note ‚Äî custom commands load at session start
- Statusline fallback ‚Äî graceful handling when `/statusline` doesn't work
- `/clear` flow fixed ‚Äî no more recursion after context reset
- CLAUDE.md loading clarity ‚Äî what auto-loads vs on-demand

## v1.0.2 ‚Äî Hooks Format Fix

Reverted incorrect object matcher format. Confirmed via official docs and Anthropic repo that matcher is always a string regex.

**Before (wrong):** `"matcher": {"tools": ["WriteTool"]}`
**After (correct):** `"matcher": "Write|Edit"`
